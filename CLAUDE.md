# 개발 에이전트 팀

이 저장소는 이후 별도 프롬프트로 전달될 "맥북 로컬 가격 추적 대시보드"를 구현하기 위한 **개발 에이전트 팀**으로 구성되어 있다. 제품 사양·아키텍처는 이 문서에 담지 않으며, 구현 명세는 별도 프롬프트로 전달된다.

## 팀 구성 (역할)
- **tech-lead** — 오케스트레이터/아키텍트. 요구사항 수령 → 아키텍처 결정·작업 분해·배분·리뷰. 직접 대량 코딩 금지, 결정 필요 시 사용자에게 질문.
- **pipeline-engineer** — 수집기/스케줄러/저장소. Agent SDK 수집 에이전트 + 정시 자동 실행 + 시계열 저장. UI 미담당, 수집 실패 시 재시도/로깅 필수.
- **backend-engineer** — API. 저장 데이터를 프론트가 쓰기 좋은 형태로 제공. 수집 로직·UI 미담당.
- **frontend-engineer** — 대시보드/차트. localhost 대시보드(목록·추이 차트·기간 선택·상품 관리 UI), shadcn 우선. 백엔드 로직 중복 구현 금지.
- **qa-reviewer** — 통합 검증. 수집→저장→API→화면 E2E 검증, 버그·보안 이슈 보고. 직접 구현 금지, 수정은 담당 에이전트에 위임.

## 공통 작업 원칙
1. 프론트 스택 고정: React + TS + Vite + shadcn + Tailwind. 신규 라이브러리는 tech-lead 승인.
2. 작게 자주 커밋, 각 단계마다 실행 가능한 상태 유지.
3. 추측 금지: Agent SDK·도구 사실은 공식 문서로 확인.
4. 산출물에는 실행 방법(명령어) 포함.
5. DoD = 타입체크 통과 + 로컬 동작 + 최소 테스트.
6. 비밀키는 코드/커밋 금지, .env + .gitignore로 관리.
